📦 子应用1 · CSS 隔离对比

在这里观察子应用侧的表现，切换到主应用首页进行对比。切换顶部沙箱模式后刷新。

A 弹窗（非 Portal）

此弹窗在 Angular 组件模板内渲染，不走 Portal，所以 experimental 模式下样式仍有效。对比子应用3的 El-Plus Dialog（走 Portal），experimental/strict 下样式会丢失。

B CSS 变量
--shared-color 实际读到：{{ sharedColor }}
子应用期望：蓝 #409eff
主应用定义：橙 #ff6600
--host-primary（主应用专属） 实际读到：{{ hostPrimary }}
若有值说明 :root 变量泄露进来了
C 类名冲突 .conflict-btn

子应用期望此按钮为蓝色。
none：被主应用 .conflict-btn { background: red !important } 污染 → 红色
experimental / strict：主应用全局样式被隔离 → 蓝色

D @keyframes spin 动画
子应用1
旋转+缩放

子应用1 定义的 @keyframes spin：旋转 + 缩放脉冲。主应用也定义了同名 spin（只旋转）。none 模式下两者可能互相覆盖；strict 模式下完全隔离，动画各自独立。

E :root 变量泄露（所有模式）

主应用 --host-primary 实际读到：{{ hostPrimary }}
无论哪种沙箱模式，子应用都能读到主应用 :root 里定义的变量，qiankun 无法阻止。
